There are some comments necessary to the former list of programming languages. Machine language commands can be processed directly by a processor. Every command is given in binary form using 0 and 1. This is per definition not natural to human beings. Here comes the assembler into play. An assembler works with the so-called mnemonics which makes progamming on processor level easier for a human being. Commands are no longer given in a combination of 0 and 1. Assembler uses e.g commands like shift arithmetic right (SAR) and shift logical right (SHR). The next level on programming languages are C/C++ which are near to the assembler programming. Other programming languages e.g. are again written in C/C++. Here we have to talk about interpreter and compiler. The programming language itself is nothing else than semantic and syntactic rules and restrictions. To bring them to live one needs a interpreter or compiler.
