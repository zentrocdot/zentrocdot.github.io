Extensions for the Web UI AUTOMATIC1111

Preface

When I talk about AI image generation, one must know that I am working with Stable Diffusion. There are a number of web UIs that can be used for the application of Stable Diffusion. Prominent ones are Easy Diffusion, AUTOMATIC1111, ComfyUI, Forge, Fooocus and InvokeAI.

After working with the web UI Easy Diffusion, I switched to the web UI AUTOMATIC1111. The reason for switching from one to another web UI is simple. I was more or less at the end of the possibilities which Easy Diffusion offered to me.

When I reached my limits in terms of AI image generation, I started to build my own models in form of Embeddings, Hypernetworks, LORAs, DreamBooth models and merged base models. Within the model creation process, I realised that I was missing Extensions to be able to work better.

I learnt how to programme such Extensions for AUTOMATIC1111. A major advantage of using AUTOMATIC1111 is exactly the ability to implement such user-programmed Extensions. With these Extensions, the capabilities of AUTOMATIC1111 can be significantly increased.

Installation of the Extensions

The installation of each Extension is as easy as it is. Go to tab Extensions and go to tab Install from URL. Put in the link from the reference and your are done.

Due to my bad personal experience with AUTOMATIC1111 contributors, I will not be making any further attempts to have my extensions added to their official list.

Final Words

It remains to be seen whether I will remain loyal to AUTOMATIC1111 in the long term. Unfortunately, one have to assume that the days of AUTOMATIC1111 are numbered, as there is no adaptation to the current versions of Gradio, for example. Various Python modules are already in conflict with each other if you keep the systems in use up to date. More problematic are the bugs of the Gradio version, which is in use by AUTOMATIC1111. These make programming the simplest Extension next to other reasons very laborious.
