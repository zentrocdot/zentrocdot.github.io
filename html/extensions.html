Extensions for the Web UI AUTOMATIC1111

Preface

When I talk about AI image generation, one must know that I am working with Stable Diffusion. There are a number of web UIs that can be used for the application of Stable Diffusion. Prominent ones are Easy Diffusion, AUTOMATIC1111, ComfyUI, Forge, Fooocus and InvokeAI.

After working with the web UI Easy Diffusion, I switched to the web UI AUTOMATIC1111. The reason for switching from one to another web UI is simple. I was more or less at the end of the possibilities which Easy Diffusion offered to me.

When I reached my limits in terms of AI image generation, I started to build my own models in form of Embeddings, Hypernetworks, LORAs, DreamBooth models and merged base models. Within the model creation process, I realised that I was missing Extensions to be able to work better.

I have learnt how to program such extensions for AUTOMATIC1111. A great advantage of using AUTOMATIC1111 is precisely the possibility of implementing such user-programmed extensions. With these extensions, the possibilities of AUTOMATIC1111 can be extended considerably.

Available Extensions

In the first 10 days that I have been working on programming in AUTOMATIC1111, one can not expect miracles. Most of the subsequent listed Extensions have be written in this time frame. The Extensions got better with the learning effect I had. Since my extensions for the AUTOMATIC1111 are not of high enough quality, as the AUTOMATIC contributors say, to say it diplomatically, I will leave the availability of the Extension via this website. I leave it to the user to judge their usefulness. The Extensions fulfil their purpose and do what they are supposed to and are a great help to me.

  • sd-webui-adjust_lora_filename
    • Adds a tab in which the LoRA tag for the output file can be checked and updated
  • sd-webui-lora_metadata_viewer
    • Adds a tab in which the LoRA metadata can be displayed, downloaded and copied
  • sd-webui-predefined_aspect_ratios
    • Extensions which is adding predefined aspect ratio buttons to the web UI
  • sd-webui-aspect_ratios-dd
    • Extensions which is adding predefined aspect ratio selectable via a dropdown menu to the web UI
  • sd-webui-calc_aspect_ratio
    • Calculates the aspect ratio from Width and Hight of the web UI
  • sd-webui-aspect_ratio_calc
    • Calculates the aspect ratio from given Width and Hight or from the given values by the web UI
  • sd-webui-aspect_ratio2width_height
    • Calculates Width and Hight from a given aspect ratio and allows taking them over to the web UI

The list of Extensions will be updated here by me if necessary.

Installation of the Extensions

The installation of each Extension is as easy as it is. Go to tab Extensions and go to tab Install from URL. Put in the link from the reference and your are done.

Due to my bad personal experience with AUTOMATIC1111 contributors, I will not be making any further attempts to have my extensions added to their official list.

Final Words

It remains to be seen whether I will remain loyal to AUTOMATIC1111 in the long term. Unfortunately, one have to assume that the days of AUTOMATIC1111 are numbered, as there is no adaptation to the current versions of Gradio, for example. Various Python modules are already in conflict with each other if you keep the systems in use up to date. More problematic are the bugs of the Gradio version, which is in use by AUTOMATIC1111. These make programming the simplest Extension next to other reasons very laborious.

It is not desired and not allowed without my explicit permission to add my Extensions that I have painstakingly written the last week over to the AUTOMATIC1111 Extension Repository.
